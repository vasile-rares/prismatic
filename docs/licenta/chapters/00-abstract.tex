This thesis presents the design and implementation of \textit{Favigon}, a full-stack web platform built to reduce the gap between interface ideas, visual editing and executable code. The project started from a practical issue that appears frequently in modern workflows: design is often created in one tool, implementation is rebuilt manually in another, and the transition between the two is slower and less precise than it should be.

The proposed solution brings together a visual canvas editor, AI-assisted generation, code export and a social area for publishing and reusing projects. From a technical perspective, the application uses Angular 20 on the frontend, ASP.NET Core 9 on the backend, PostgreSQL for persistence and Docker Compose for service orchestration. Controlled external exposure is handled through Cloudflare Tunnel.

The central component of the platform is the canvas editor. It supports multi-page work, selection, gestures, history, auto-save and preview. In order to avoid coupling the editor directly to the generated code, the project introduces an intermediate UI representation that is reused by both the editor and the converter engine. This decision makes validation and export more predictable.

The AI component does not generate final code directly. Instead, the platform uses a three-phase pipeline: intent extraction, structure generation and styling. The result is validated, can be repaired when it does not match the expected schema, and is then converted into HTML, React or Angular output. This staged approach improves control over the generated result and makes errors easier to trace.

The thesis also covers the product capabilities required for a usable platform: JWT-based authentication through cookies, OAuth login, password reset, two-factor authentication, project management, public and private projects and social actions such as like, star, follow and fork. For validation, the backend and converter are covered by 108 passing automated tests, while the main user flows were also checked through functional scenarios.

The main contribution of the work lies in the coherent integration of the canvas editor, the intermediate UI model, the AI pipeline and the multi-framework export engine into a single application. The result is not positioned as a replacement for mature commercial tools, but as a functional software platform that demonstrates a controllable and explainable design-to-code workflow.
